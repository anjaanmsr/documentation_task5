Pkg_Task5

Introduction

We have simulated an automated warehouse in Gazebo consisting of UR5#1 and UR5#2 shipping packages of various priorities from the warehouse. The warehouse receives orders from the customer according to which the UR5_1 arm will pick the packages according to their priorities, high priority will be picked up first and placed on the conveyor belt followed by medium priority and then the lower priority packages. The UR5_2 Arm will pick the objects from a moving conveyor belt and segregate them according to the color corresponding bins. When the packages are placed into the bins, an alert email is sent using a ROS- IoT bridge to the customer about the shipment of packages from the warehouse

Implementation

We have created two different packages in order to accomplish the given task.

pkg_ros_iot_package and pkg_task

pkg_ros_iot_package:

This Package has internet connection and it connects to internet to push data to the Inventory Management System and fetch data from the Inventory Management System.

pkg_task5 request pkg_ros_iot_package to complete its requirements of internet connection.

  1. msg
    • msgMqttSub.msg
  1. scripts
    • iot_client: This Module connects us with the Iot Server.
    • pyiot: This module build connection with mqtt client.
    • node_action_server_ros_iot_bridge.py: This module helps in creating server which send data to sheet using mqtt.
    • node_place_online_order: This module sends incoming order.

pkg_task5

This package controls both ur5_1 and ur5_2, ur5_1 picks boxes from shelf and place on the belt and ur5_2 picks boxes from the belt and place in the bin.

structure

  1. config
    • task5_bin.scene
    • task5_pod.scene.
  1. launch
    • task5.launch
  1. msg
    • iotToBot1.msg
    • OrderDetailArray.msg
    • serviceHandle.msg
  1. scripts
    • iot_client: This Module connects us with the Iot Server.
    • handling_ros_service: This module build connection with mqtt client.
    • ur5_moveit: This module configures rviz planning scene and ur5.
    • camera_one : This module provides details of boxes in the shelf.
    • ur5_1_controller : Controls ur5_1 picking and placement operations.
    • ur5_2_controller: Controls ur5_2 picking and placement operations and belt operations
  1. CMakeList.txt
  1. package.xml
